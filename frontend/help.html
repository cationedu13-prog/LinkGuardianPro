Getting Started

How do I get started?
1. Create a free account at linkguardian.pro/signup
2. Add up to 5 links to monitor (free plan)
3. Set up your Telegram Chat ID for alerts (optional but recommended)
4. Links are checked automatically twice daily (6 AM and 6 PM UTC)
5. You'll receive Telegram alerts instantly when a status changes
How do I find my Telegram Chat ID?
Follow these steps to get your Telegram Chat ID:

1. Open Telegram and search for @userinfobot
2. Start a chat and send /start
3. The bot will reply with your Chat ID (a number like 123456789)
4. Copy this number and paste it in Dashboard → Settings → Telegram Chat ID

Alternatively, search for @RawDataBot and it will show your chat ID when you send any message.

Important: You must also start a chat with our bot first. Our Telegram bot username will be shown in your dashboard settings page.
What does "multi-layer" detection mean?
LinkGuardian uses three detection methods in sequence, falling back to the next if the previous fails:

Layer 1 – Direct HTTP: Fastest method. Makes a direct HTTP request to the URL with browser-like headers. Works for most simple websites and some e-commerce sites. Typically completes in under 5 seconds.

Layer 2 – ScraperAPI: Uses ScraperAPI's infrastructure to render JavaScript-heavy pages. More effective for sites like Amazon and Flipkart that block direct requests. Requires ScraperAPI key to be configured.

Layer 3 – Playwright: Uses a full headless Chromium browser as the final fallback. Most accurate but slowest. Can handle any modern website but requires more server resources.

Link Monitoring

How often are my links checked?
Free and Pro plans: Twice daily — approximately 6 AM and 6 PM UTC (11:30 AM and 11:30 PM IST).

Pro, Business, Agency plans: Can be set to hourly frequency per link via the Add Link modal.

You can also trigger a manual check any time from your dashboard by clicking the "Check" button next to any link.
Why am I getting false "Out of Stock" alerts for my product?
Out-of-stock detection uses keyword matching on the page HTML. False positives can occur when:

• The product page contains out-of-stock text for related products or ads on the same page
• The website uses non-standard wording not in our keyword list
• The website is geo-restricted and shows different content to our servers
• Amazon or Flipkart returns a CAPTCHA page instead of the product page
• The page is dynamically loaded and our checker sees an incomplete version

If you're getting consistent false positives for a specific URL, try using the "Check Now" button to see the current result and compare with what you see in your browser. You can also contact us to report persistent false positives so we can improve our keyword patterns.
Why does my Amazon link show as "Broken" even though it works in my browser?
Amazon aggressively blocks automated requests. When Layer 1 (direct HTTP) is blocked, it may appear as an error or broken. Solutions:

1. Enable ScraperAPI: Set up a ScraperAPI key for your account (Layer 2) — this bypasses most Amazon blocks
2. Use a different URL format: Some Amazon URL formats are more bot-friendly than others
3. Check the layer used: If the result shows "layer1", it means Layer 2/3 weren't available. Upgrade your plan or configure ScraperAPI

Note: Monitoring Amazon links is challenging due to their anti-bot systems. Results may vary. See Terms of Service for our accuracy limitations.
What happens if I get too many false alerts?
Our system sends alerts only once per status change — we don't spam you. If a link flaps between statuses (keeps changing), you'll get one alert per change.

If you're getting false alerts frequently for a specific platform, you can:
1. Delete the link and re-add it with a different platform setting
2. Contact our support to report the issue
3. Check if the URL itself has changed (use the free checker tool first)

Plans & Billing

What happens when my plan expires?
When your paid plan expires:

1. Your account automatically reverts to the Free tier (5 link limit)
2. Link monitoring is paused for links above the Free tier limit — they are NOT deleted
3. You'll receive a Telegram alert at 7, 3, 1 days before expiry and on the day it expires
4. When you renew, monitoring resumes for all your links up to your new plan limit

Your data is safe. We never delete your links or history due to plan expiry.
Is my data safe?
Yes. We take data security seriously:

• Passwords are hashed using SHA-256 — we never store plain-text passwords
• All communication is encrypted via HTTPS
• We use Supabase (PostgreSQL) hosted on secure infrastructure
• Session tokens are randomly generated and expire after inactivity
• We do not sell your data to third parties
• Check history is automatically deleted after 90 days

See our Privacy Policy for full details.
Can I export my link data?
Currently, you can view your links and check history in the dashboard. Export functionality (CSV download) is planned for a future update. Contact us if you need an urgent data export.

Technical Questions

The free checker shows a different result than my dashboard check. Why?
The free checker always uses Layer 1 only (direct HTTP) and has a rate limit. Dashboard checks use all available layers based on your plan. This means dashboard checks are generally more accurate, especially for complex sites.
What is the free checker rate limit?
The free checker (on the homepage) allows up to 10 checks per hour per IP address. This limit exists to prevent abuse. Creating a free account gives you unlimited manual checks from the dashboard.
